# OTN 光缆中断统计地图性能优化需求

## 1. 问题背景与定位

**现象：** 用户反馈 `/plugins/otnfaults/statistics/` 页面中，点击"地图"按钮弹出的地图窗口有时很卡，系统 GPU 占用率异常增高。故障点仅约 10 个，系统中有约 1000 个站点。

**根因分析：**

卡顿并非由 10 个故障点引起，而是因为 1000 个站点的标签渲染启用了三个 GPU 密集型特性叠加造成的：

| GPU 杀手 | 说明 |
|----------|------|
| `text-halo-blur: 1` | 对 1000 个文字标签逐个施加高斯模糊着色器，是最严重的开销来源 |
| 碰撞避让算法 | `text-allow-overlap: false` + `text-optional: true`，每帧重算 1000 标签位置 |
| 圆点描边 | `circle-stroke-width: 1`，1000 个圆点额外绘制描边 |

再叠加 **iframe 上下文**（与父页面 ECharts 争夺 GPU）+ 省界 GeoJSON + OTN 路径 PMTiles，就造成了"有时卡有时不卡"的偶发特征。

## 2. 优化方案

### 2.1 站点图层性能降档（核心）
在统计地图模式插件 `statistics_cable_break_mode.js` 中，初始化后对共享站点图层的 GPU 密集属性进行降档：
- 移除 `text-halo-blur`（高斯模糊 → 0）
- 提高站点标签的 `minzoom`（减少低缩放级别碰撞计算量）
- 简化圆点渲染（去描边、固定透明度）

### 2.2 故障点渲染重构
将现有 DOM Marker（`maplibregl.Marker`）替换为 WebGL Circle Layer，消除 DOM 回流开销。

### 2.3 不改动后端
`sites_data` 保留原样返回，前端通过图层属性优化解决性能问题。

## 3. 影响范围
所有改动仅限于 `statistics_cable_break_mode.js`。不影响主故障分布图等其他地图模式。

## 4. 验收标准
1. 打开光缆中断统计地图后拖拽/缩放流畅，GPU 占用率回落至正常水平
2. 1000 个站点正常显示（圆点 + 放大后标签）
3. 故障点悬浮弹窗、点击交互行为保持不变
