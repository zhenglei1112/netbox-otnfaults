# 【PRD】一张图“割接可视化与管理”功能需求与设计

## 1. 业务背景与目标
割接（Cutover）是光传送网（OTN）日常维护中高风险且核心的操作。一个割接任务可能涉及多个光缆路由的调整，并会影响多个承载的业务。
为了提高运维人员对网路调整的感知能力，并实现对割接时间、地点的精细化管理，本项目拟在现有的**“一张图（unified_map）”**功能中引入割接的可视化渲染，直观展示割接的地理位置、时间进度、影响范围，并提供动态时间范围和状态类型的过滤，协助运维人员一目了然地把控全网即将发生的割接计划。

---

## 2. 核心功能需求

### 2.1 地理定位与上图机制
- **数据源**：割接数据来源于插件中已有的 `CutoverTask` 模型。
- **定位算法**：
  1. 优先使用割接自身记录的经纬度 `cutover_longitude` 和 `cutover_latitude`。
  2. 若自身经纬度为 `null`，则退而求其次，定位到该割接的 A端站点 `interruption_location_a` 的经纬度。
  3. 若以上两者均无有效的经纬度，则在地图上不予渲染该割接点。
- **数据加载**：支持在地图数据视图 `OtnFaultMapDataView` 中，通过查询参数动态请求满足条件的割接数据。

### 2.2 割接筛选控制面板
在地图左侧的 `LayerToggleControl`（图层控制）或单独新增的割接控制面板中，提供两层维度的动态过滤：
1. **割接类型（状态）过滤**：
   - 支持勾选：待实施、申请中、已完成、被取消。
   - **默认行为**：默认显示“待实施”状态。
2. **割接时间范围筛选**：
   - 提供下拉菜单或快捷标签切换：
     - **未来24小时**（关注临近割接）
     - **未来7天**（默认）
     - **未来30天**（月度计划）
     - **过去30天**（回顾）
     - **全部时间**

### 2.3 视觉与动效设计（符合 Rich Aesthetics 规则）
- **割接图标设计 (SVG)**：
  - 设计精致的割接专属图标（光纤插头与齿轮、扳手结合，或双向切换的网络隐喻图标）。
  - 针对不同状态分配 curated 高级配色：
    - **待实施 (`pending_implementation`)**：金橙/琥珀色 (`#F5A623`)，象征即将动工。
    - **申请中 (`applying`)**：科幻极客蓝 (`#0D6EFD`)，象征审核流转。
    - **已完成 (`completed`)**：现代翡翠绿 (`#198754`)，象征顺利完工。
    - **被取消 (`cancelled`)**：低饱和中性深灰 (`#6C757D`)，降低视觉存在感。
- **即将割接的“呼吸灯”动效**：
  - 对于**未来 24 小时内**即将实施的待实施割接，在割接点图标下方添加一个**发光光晕层**。
  - 使用 MapLibre Circle 图层，实现其半径和透明度交替变大变小的**呼吸灯闪烁（Ripple/Pulse Effect）**，增强视觉警示，提醒运维人员重点关注。

### 2.4 信息卡片（Popup）与交互设计
- **卡片悬停展示**：鼠标悬停在割接图标上时，显示专门定制的割接信息 Popup。
- **核心展示字段**：
  - **头部**：【割接计划】 + 割接编号 `cutover_no` + 精致的状态 Badge。
  - **时间进度条/倒计时（亮点功能）**：
    - 若为“待实施”，动态计算距离割接开始的倒计时（例如：“*距离割接开始：2天15小时*”）。
    - 若为“待实施”且当前时间**已超过**计划开始时间但状态未更新，以**红色闪烁**文字警示：“*已超时 5 小时未实施*”。
    - 若正在实施（处于割接时间窗口内），显示“*割接正在进行中 (历时: XX)*”。
  - **割接原因**：精炼展示 `cutover_reason`。
  - **割接具体地点**：展示 `cutover_location`。
  - **影响范围**：展示 A端站点及受影响的 Z端站点列表。
  - **影响业务**：列出受影响的重要业务数量，支持点击查看明细。
- **高亮联动**：
  - 悬浮在割接点上时，地图自动联动，将关联的 A-Z 端站点以及在该区间内受影响的 **光缆路径（OtnPath）** 加粗并呈现**流光流动动效**，直观呈现空间影响面。

## 3. 技术方案设计

### 3.1 后端接口与序列化
在 `netbox_otnfaults/services/fault_map_data.py` 中，定义 `CutoverMapMarkerSerializer` 对割接模型进行包装序列化，并新增过滤逻辑：
```python
class CutoverMapMarkerSerializer:
    def __init__(self, cutover: CutoverTask):
        self.cutover = cutover

    def data(self) -> dict | None:
        # 实现定位兜底、状态识别、时间格式化、关联影响站点/路径/业务的数据打包
```
在 `views.py` 的 `OtnFaultMapDataView` 中，接收前端过滤参数 `cutover_status` 与 `cutover_time`，动态查询数据库并将割接的 markers 打包返回给前端。

### 3.2 前端图层与控件 (Unified Map Integration)
- **图层注册**：
  在 `fault_mode.js` 中注册三个新图层：
  - `cutover-points-glow` (圆形图层，提供发光呼吸特效，根据是否即将发生动态控制不透明度)。
  - `cutover-points` (符号图层，渲染定制的 SVG 割接图标)。
- **图层开关与过滤面板**：
  - 修改 `LayerToggleControl.js`。在“其他图层”中增加一个“显示割接计划”勾选项。
  - 在勾选后展开一个精心设计的“割接过滤器”卡片面板（`CutoverFilterPanel`），可以无缝悬浮在地图边缘，提供高级时间滑块与状态多选框。

## 4. 实施阶段计划

1. **第一阶段：后端数据源整备**（开发 `CutoverMapMarkerSerializer` 并注入地图 AJAX API 支持参数化过滤）。
2. **第二阶段：前端图层渲染与动效**（在地图中渲染割接 Symbol、加入 24 小时呼吸光晕特效图层）。
3. **第三阶段：交互控件定制**（开发精致的割接时间/状态控制过滤器、定制优雅的信息 Popup 模板及流光高亮联动）。
4. **第四阶段：验证与调优**（在无真 Netbox 环境下，通过测试脚本或本地网页资源，配合详细的 Mock 离线数据进行完整验证）。
