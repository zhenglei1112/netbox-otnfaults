# 故障统计 - 业务故障统计功能需求

## 1. 背景

在现有故障统计（物理故障统计）基础上，增加**业务故障统计**分析维度，以 `OtnFaultImpact`（故障影响业务）为统计对象，按业务聚合展示各项关键指标。

## 2. 功能说明

### 2.1 页面结构

- 复用现有表头时间选择器（按年/月/周）和时间范围显示
- 在时间选择器下方引入 **Bootstrap 5 Tab** 切换
  - Tab 1: **物理故障统计**（原有全部内容）
  - Tab 2: **业务故障统计**（新增）

### 2.2 业务故障统计 Tab

- 以每个业务（BareFiberService / CircuitService）为维度
- 每个业务使用一张**统计卡片**展示以下指标：

| 统计项 | 说明 |
|--------|------|
| 故障总数 | 业务受影响次数，含分类明细（中断x次、抖动x次、劣化x次） |
| 累计总时长 | 所有事件的业务中断时长之和（小时） |
| 平均单次时长 | 累计总时长 / 故障总数 |
| 长时故障 | 业务中断 ≥6h 的次数 |
| 重复故障 | 同一业务 60 天内再次受影响的次数 |
| 可用率(SLA) | `(周期总小时 - 累计中断) / 周期总小时 × 100%` |

### 2.3 SLA 计算规则

- 周期总小时数基于筛选的时间范围（如按年=8760h、按月=720h等）
- 当一个业务在同一时段有多次故障时，合并重叠时段避免重复计算不可用时长
- SLA ≥ 99.99%：绿色；≥ 99.9%：黄色；≥ 99%：橙色；< 99%：红色

## 3. 后端 API

- 路由: `statistics/service-data/`
- 接受与物理故障相同的时间参数
- 返回按业务聚合的统计 JSON 数据

## 4. 交互说明

- Tab 切换时按需加载对应数据
- 时间选择器变更时刷新当前活动 Tab 数据
- 业务卡片按故障总数降序排列
