# OTN 全国网络故障态势大屏 - 坐标对齐与渲染优化需求文档 (PRD)

## 1. 背景与目标
在大屏展示系统中，原有故障点标记（HTML Marker）在某些屏幕缩放比例或 CSS 布局下，会与基于 WebGL 渲染的站点、路径及省界层产生明显的地理位置偏移。
本项目的目标是统一全屏地图的渲染管线，消除由于 DOM 层与 WebGL 层投影不一致导致的经纬度对齐偏差。

## 2. 核心痛点
- **HTML Marker 偏移**: `maplibregl.Marker` 使用 CSS 像素定位，在大屏 CSS `scale` 缩放或复杂的绝对布局下，其投影计算与 WebGL 核心产生的像素位置可能存在数个像素乃至数十公里的偏差。
- **视觉不一致**: 站点和路径是动态渲染的 WebGL 元素，与 HTML 标记在缩放、旋转时的跟随速度不完全同步。

## 3. 解决方案
### 3.1 渲染模式转换
- **废弃 HTML Marker**: 全面移除 `renderFaultMarkers` 中基于 `div` 元素的 HTML Marker 实现。
- **启用 WebGL Circle Layer**: 将故障标记改为 MapLibre 的原生图层（Layer）。使用三层叠加结构模拟原有的脈冲效果：
    1. `faults-pulse`: 动态脉冲外环（动画驱动半径与透明度）。
    2. `faults-glow`: 静态光晕层。
    3. `faults-core`: 核心亮点。

### 3.2 动画引擎优化
- **移除 CSS 动画**: 不再依赖 CSS `@keyframes`，改为在 `map_engine.js` 的 `requestAnimationFrame` 循环中直接更新 WebGL 属性。
- **亚像素同步**: 确保脉冲效果与地图缩放、平滑飞行时绝对实时跟随，无任何肉眼可见的延迟或漂移。

## 4. 坐标逻辑定义
- **统一标准**: 全系统（后端 API、站点数据、故障数据）统一使用 **WGS-84** (GPS) 坐标系。
- **省界转换**: 对于来源于 GCJ-02（火星坐标系）的省界 GeoJSON，在前端页面加载时进行实时转换，确保其与 WGS-84 数据流对齐。

## 5. 验收标准
1. 所有故障脉冲标记中心点必须与其归属的站点（蓝色圆点）完全重合。
2. 聚焦故障时的 Camera 飞行（FlyTo）结果必须使标记处于镜头正中心。
3. 地图在旋转或倾斜（Pitch/Bearing）时，故障标记不得离开站点位置。
