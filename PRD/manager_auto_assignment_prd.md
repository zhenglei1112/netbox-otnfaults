# PRD - Province-based Manager Auto-Assignment

## 变更背景
为了提高故障录入效率，根据官方分工地图，实现选择省份后自动匹配对应的线路主管。

## 业务映射规则
| 线路主管 | 负责省份 (系统匹配关键字) |
| :--- | :--- |
| **冯鑫源** | 甘肃、宁夏、青海、新疆 |
| **姜川** | 广东、湖南、福建、重庆、贵州、云南 |
| **李立彬** | 北京、天津、河北、河南、山西、广西 |
| **孙振伟** | 湖北、安徽、江苏、上海 |
| **李小涛** | 黑龙江、吉林、辽宁 |

## 实现方案
1.  **前端监听**: 在故障编辑页 (`otnfault_edit.html`) 监听 `province` (省份) 字段的变更。
2.  **名称提取**: 获取当前选中项的文本（如“河池”中的“广西”，或“广东省”）。
3.  **关键字匹配**: 检查文本是否包含上述映射表中的关键字。
4.  **异步查找**: 调用系统 API `/api/users/users/?q=[主管姓名]` 获取用户 ID。
5.  **自动填充**: 将查询到的用户自动填入 `line_manager` (线路主管) 字段。

## 注意事项
- 匹配逻辑使用 `includes`，因此“广东”可以正确匹配“广东省”。
- 如果 API 查询不到对应姓名（如系统未创建该用户），则不会进行自动填充。
- 手动修改：自动填充后，用户仍可手动更改线路主管。
