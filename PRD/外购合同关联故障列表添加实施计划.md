# 外购合同关联故障列表添加实施计划

## 1. 需求分析
**目标**：在 `netbox-contract` 插件的合同（`Contract`）模型详情页中，添加关联的 OTN 故障列表信息。
**背景**：目前系统的 `OtnFault` 模型中已经包含 `contract` 字段（指向 `netbox_contract.Contract`），需要在查阅外购合同（代维/租赁合同）时，能够直观地看到因此合同提供服务而关联的故障列表。

## 2. 设计方案
使用 NetBox 提供的 `PluginTemplateExtension` 机制：
1. **修改 `template_content.py`**：
   - 增加一个 `ContractOtnFaults` 类，继承自 `PluginTemplateExtension`。
   - 配置 `models = ['netbox_contract.contract']`。
   - 重写 `full_width_page` 方法。
   - 在 `full_width_page` 中，获取当前合同对象 (`obj = self.context['object']`)。
   - 查询 `OtnFault.objects.filter(contract=obj)` 得到相关联的故障。
   - 使用已有的 `OtnFaultTable` 对查询结果进行渲染。
   - 按照代码规范添加自定义的主题及分页支持。
   - 将该模板扩展添加到 `template_extensions` 列表中。

2. **创建相关前端模板文件**：
   - 创建 `templates/netbox_otnfaults/inc/contract_otn_faults.html` 模板文件。
   - 模板中包含表格渲染逻辑：自定义分页布局（根据 AGENTS.md 规范：禁止使用自动分页，使用自定义组件），并渲染传入的 `faults_table`。

## 3. 任务清单 (Task List)
- [ ] **Task 1**：创建文件 `netbox_otnfaults/templates/netbox_otnfaults/inc/contract_otn_faults.html` 并实现 NetBox 风格的面板包装以及按照要求禁止默认分页，应用自定义的分页 UI。
- [ ] **Task 2**：修改 `netbox_otnfaults/template_content.py`，新增 `ContractOtnFaults` 扩展，实例化 `OtnFaultTable` 并在渲染前执行分页和 `RequestConfig` 配置。
- [ ] **Task 3**：完成审查代码，确认没有引用 NetBox 不支持的库或视图。

## 4. 思考过程 (Thought Process)
1. **验证关联关系**：确认 `OtnFault` 模型上有 `contract` 字段。
2. **选择拓展区域**：外购合同下挂的故障可能会很多且表格列比较多（日期、站点、代维方等），所以使用 `full_width_page` 方法进行横宽展示最为合适。
3. **分页处理**：考虑到同页面下可能有多个表格分页的风险，我们可以在分页时针对 `OtnFaultTable` 将分页参数名称设置为 `faults_page`。
4. **渲染规范遵循**：遵守《AGENTS.md》规定的要求，隐藏原始分页控件（`<style>.table-container ul.pagination { display: none !important; }</style>`），在模板末尾循环输出页码信息。
