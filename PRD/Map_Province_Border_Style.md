# Netbox OTN 可视化插件 - 故障分布图省界颜色样式调整计划

## 1. 需求说明
将故障分布图模块中加载的省界（GeoJSON）数据的颜色调得更明显一些，但同时要确保其视觉层级不能高于业务数据（线路 `#00cc66` / `#3B82F6` 系列和站点 `#00aaff` 等）。核心目的是提供清晰可辨的省级边界作为地理参考，又不产生视觉上的抢戏。

## 2. 涉及文件及修改范围
- `netbox_otnfaults/static/netbox_otnfaults/js/dashboard/map_engine.js` (大屏模式)
- `netbox_otnfaults/static/netbox_otnfaults/js/unified_map_core.js` (基础统一地图核心)

## 3. 设计与实现步骤

### 第 1 步：分析现有配色系统与需求
- **业务线路**：透明度约为 0.6 ~ 0.8 的鲜亮色（如亮绿 `#00cc66`、亮蓝 `#3B82F6`）。
- **业务站点**：高亮青色 `#00aaff` 或带有发光的标记点。
- **现有省界**：
  - `map_engine.js`：原本使用亮青色线 `rgba(0, 210, 255, 0.4)`，过于饱和容易与业务线混淆。
  - `unified_map_core.js`：原本使用半透明白色 `rgba(255, 255, 255, 0.6)`，亮度偏高而色彩融入度不够。
- **目标配色策略**：使用灰蓝色调（如 `rgba(90, 140, 190, 0.7)`）。灰蓝色提供了足够的冷色调对比度让用户一眼看清，同时较低的饱和度可以使其稳定居于底层，甘当绿叶。

### 第 2 步：调整 `map_engine.js` (大屏 3D 控制引擎)
在 `_loadProvinceLayer` 函数中修改省界的 `line-color` 和 `line-width`：
- 修改顶面边界线颜色为 `rgba(100, 130, 170, 0.7)`。
- 修改底层发光辅助线颜色为 `rgba(100, 130, 170, 0.25)`。

### 第 3 步：调整 `unified_map_core.js` (统一地图核心引擎)
在 `_initSharedLayers` 的自定义 GeoJSON 底层加载时，修改 `user-geojson-line` 图层：
- 颜色由原来的半透明白修改为蓝灰色 `rgba(90, 140, 190, 0.7)`。
- 线宽修改至 1.5 使形态更为清晰。

## 4. 验证方式
- 检查代码替换是否精准覆盖省界（`user-geojson-line` 和 `province-border-top` 等图层）。
- 无需执行后端命令，通过浏览器直接刷新访问地图界面并加载省界 GeoJSON 数据，人工验证颜色是否存在并处于预期的偏淡灰蓝状态。
