# 态势感知大屏面板压缩问题修复需求

## 1. 需求背景与问题分析
- **当前表现**：态势感知大屏左上角的“网络健康度”栏在下方栏目条目较多时会被压缩，导致其中的活跃故障等小字信息无法完整显示。
- **根本原因**：大屏左侧面板（`.side-panel`）是一个采用 `flex-direction: column` 布局的 Flex 容器，内部的各个卡片组件（如 `.panel-card`）默认具有 `flex-shrink: 1` 属性。当容器内元素总高度超出容器高度时，各个卡片会自动发生收缩以适应容器，导致内容显示异常。

## 2. 实施方案
- **修改文件**：`netbox_otnfaults/static/netbox_otnfaults/css/dashboard.css`
- **修改内容**：在提取的大屏统一样式类 `.panel-card` 中，强制添加 `flex-shrink: 0;` 属性。
- **预期效果**：强制面板卡片保持其原有高度不被压缩，当卡片总高度超出屏幕时，由于父容器（`.side-panel`）已设置 `overflow-y: auto;`，将会自动出现滚动条，从而既保证所有卡片内容显示完整，又保证布局结构不被破坏。
