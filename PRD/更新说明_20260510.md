# Netbox OTN 故障管理插件更新说明 (自 2026年5月6日)

## 一、新功能 (New Features)

1. **故障管理补充状态字段**
   - **说明**: 故障记录新增了“挂起”状态字段，便于管理特殊情况的故障流转。
   - **操作入口**: 故障列表 -> 添加/编辑故障 -> “挂起”复选框。

2. **供电故障专题字段扩充**
   - **说明**: 针对供电类故障大幅扩展了详细信息登记表单，新增“全中断”、“设备托管”标记，以及割接报告、响应措施、数据提供方、根因分析、整改情况等信息。
   - **操作入口**: 在添加/编辑故障时，若故障原因选择为“供电故障”，相关补充表单项会自动呈现。

3. **故障影响业务精细化登记**
   - **说明**: 故障影响业务关联记录中新增“业务中断状态”选项（中断/未中断），能够明确区分不同受波及业务的具体受损情况。
   - **操作入口**: 添加或编辑故障影响业务时可见状态选择下拉框。

4. **电路业务扩展信息与批量导入**
   - **说明**: 电路业务模块新增了多项扩展信息字段支持，并配套提供了 Excel 批量导入脚本。
   - **操作入口**: 
     - 页面端：电路业务详情页/编辑页。
     - 脚本端：系统菜单 `Scripts` -> 选择 `import_circuit_service_extra_fields`。

5. **故障统计大屏功能扩充**
   - **说明**: 
     - 裸纤故障卡片新增每月的“SLA”指标展示。
     - 增加了图表维度切换功能，支持按“故障数量”与“故障时长”自由切换查看统计视图。
   - **操作入口**: 统计仪表盘界面相关卡片头部的操作按钮。

6. **自动化部周报生成脚本**
   - **说明**: 新增面向部级周报的 88 类故障统计脚本，可快速自动生成所需周报数据。
   - **操作入口**: 系统菜单 `Scripts` -> 选择 `ministry_weekly_fault_report`。

7. **OTN 拓扑地图用户偏好记录**
   - **说明**: 增加了地图样式的偏好记录功能，系统可以记住并自动应用用户上次操作的地图视图及图层显示配置。

## 二、功能修正 (Bug Fixes)

1. **地图可视化 API 请求 403 权限异常修复**
   - **说明**: 修复了地图前端组件通过 API 异步请求路径和设备元数据时，因丢失 CSRF Token 和会话认证而引发的 403 Forbidden 错误，保证了拓扑和业务路由的高亮显示恢复正常。

2. **地图站点图标闪烁与异常消失修复**
   - **说明**: 修复了在交互地图上点击聚焦（Focus）或取消聚焦故障线路时，由于旧版 MapLibre GL 的表达式语法兼容性导致的站点 Icon 异常丢失与画面重绘闪烁问题。

3. **统计图表指标说明文案与排版修正**
   - **说明**: 修正了统计大屏底部相关指标说明的描述性文本，调整了对应的排版 UI 以防止挤压变形。

## 三、性能改进 (Performance Improvements)

1. **统计大屏视觉规范与体验优化**
   - **说明**:
     - 对统计大屏进行了全局 CSS 样式重构，采用基于 CSS 变量的间距（`gap`）策略，统一了各卡片组件的垂直和水平间距。
     - 将图表控制按钮统一移至卡片头部区域，对齐整体界面的视觉风格。
     - 优化了裸纤业务卡片的渲染逻辑，即便当期无故障发生，卡片也会保留标准框架占位以防止页面高度剧烈跳变。
     - 修改了业务日历热力图的展示颜色阶梯，并在图表元素上直接标注了关键的指标数值，增强了数据的可读性。

## 四、弃用 (Deprecations)

- **暂无**: 本阶段更新未对既有数据结构、API和核心业务流程进行不兼容废弃。

## 五、其他变化 (Other Changes)

1. **代码质量审计与重构规划**
   - **说明**: 对插件全量代码执行了全面的架构审查和质量审计，针对发掘出的潜在性能瓶颈、安全风险以及与 NetBox 4.x 不兼容的设计输出了改进规划方案。

2. **底层测试覆盖率稳步提升**
   - **说明**: 针对近期的重要业务变更（数据结构改造、脚本、复杂的前端地图高亮与 PMTiles 交互逻辑），均全面覆盖了对应的单元及集成测试，提高了插件项目的整体鲁棒性。
