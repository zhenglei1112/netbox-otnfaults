# 割接生成故障实施方案

## 1. 背景与目标

割接管理已经具备 `CutoverTask` 主记录和 `CutoverImpact` 影响业务记录。故障管理具备 `OtnFault` 主记录和 `OtnFaultImpact` 影响业务记录。两套影响业务模型字段高度一致，因此可以基于一条割接记录生成一条故障记录，并把割接下的影响业务同步生成到故障影响业务中。

本功能采用“预填确认”方式：用户在割接详情页点击“生成故障”，系统进入确认页面，展示将要创建的故障字段和影响业务列表。用户确认后，系统在事务中创建 `OtnFault` 和对应的 `OtnFaultImpact` 记录。

目标：
- 从 `CutoverTask` 预填 `OtnFault`。
- 从 `CutoverImpact` 批量生成 `OtnFaultImpact`。
- 在 `OtnFault` 中新增结构化来源关联 `source_cutover_task`，支持从故障反查割接、从割接反查已生成故障、防重复生成。
- 不映射 `rectification_*` 字段，故障整改闭环由故障侧独立维护。
- 保持 NetBox 原生 Bootstrap 5 风格，不引入 React/Vue，不修改 NetBox 核心目录。

## 2. 业务流程

### 2.1 用户入口

在割接详情页 `cutovertask.html` 增加“生成故障”按钮，链接到：

```text
/plugins/netbox-otnfaults/cutovers/<pk>/generate-fault/
```

页面只在用户具备创建故障权限时显示。若当前割接已经生成过故障，按钮旁显示已生成故障链接，并将按钮改为“查看已生成故障”或禁用重复生成入口。

### 2.2 预填确认页面

确认页面展示三块内容：

1. 来源割接摘要：割接编号、状态、计划割接时间、省份、A/Z 站点、割接地点、割接原因。
2. 将生成的故障信息：故障分类、起始时间、恢复时间、报备情况、资源属性、线路主管等。
3. 将生成的影响业务：按 `CutoverImpact` 列表展示业务类型、业务名称、业务影响、中断时间、恢复时间。

用户可以在确认页面调整有限字段：
- `duty_officer`
- `fault_category`
- `urgency`
- `fault_status`
- `interruption_reason`
- `interruption_reason_detail`
- `cutover_report_status`
- `cutover_report_time`
- `fault_occurrence_time`
- `fault_recovery_time`
- `closure_time`
- `comments`

不在确认页提供整改字段。

### 2.3 提交创建

用户确认后，后端使用 `transaction.atomic()`：

1. 校验割接是否已经生成过故障。
2. 创建 `OtnFault(source_cutover_task=cutover)`。
3. 复制割接 Z 端站点到故障 `interruption_location`。
4. 遍历 `cutover.impacts.all()` 创建 `OtnFaultImpact`。
5. 对裸纤影响业务复制 `service_site_z` M2M。
6. 跳转到新故障详情页。

如果割接没有影响业务，仍允许创建故障，但确认页面必须显示提示：

```text
当前割接没有维护影响业务，将只生成故障主记录。
```

## 3. 数据模型设计

### 3.1 `OtnFault` 新增结构化关联

修改 `netbox_otnfaults/models.py`，在 `OtnFault` 中新增：

```python
source_cutover_task = models.ForeignKey(
    to='CutoverTask',
    on_delete=models.SET_NULL,
    related_name='generated_faults',
    verbose_name='来源割接',
    blank=True,
    null=True,
)
```

设计说明：
- `SET_NULL`：删除割接时不删除故障，保留故障记录。
- `related_name='generated_faults'`：从割接详情页可以查询已生成故障。
- 不设置唯一约束，数据库层允许特殊情况下一个割接生成多条故障；业务流程默认阻止重复生成。若后续确认永远一割接一故障，可再增加条件唯一约束。

### 3.2 迁移

新增迁移文件：

```text
netbox_otnfaults/migrations/0075_otnfault_source_cutover_task.py
```

只修改插件目录，不修改 NetBox 核心目录。

### 3.3 API 与过滤

更新：
- `netbox_otnfaults/api/serializers.py`
- `netbox_otnfaults/filtersets.py`
- `netbox_otnfaults/forms.py`
- `netbox_otnfaults/tables.py`

要求：
- `OtnFaultSerializer.fields` 增加 `source_cutover_task`。
- `OtnFaultFilterSet` 支持按 `source_cutover_task` 过滤。
- `OtnFaultFilterForm` 增加来源割接筛选字段。
- `OtnFaultTable` 可增加“来源割接”列，但不应放到 `actions` 后面。若列表空间不足，第一阶段可只在详情页展示，不进默认表格。

## 4. 字段映射规则

### 4.1 主记录映射

| `CutoverTask` | `OtnFault` | 规则 |
| --- | --- | --- |
| `id` | `source_cutover_task` | 结构化关联，直接设置 |
| `cutover_no` | `comments` / `fault_details` | 不写入 `fault_number`，故障编号继续自动生成 |
| `registrant` | `duty_officer` | 默认使用当前用户；若当前用户不可用则用 `registrant` |
| `planned_cutover_time` | `fault_occurrence_time` | 优先使用；为空则使用 `started_at`；仍为空则使用当前时间 |
| `completed_at` | `fault_recovery_time` | 有值则预填 |
| `closed_at` | `closure_time` | 有值则预填 |
| `province` | `province` | 直接映射 |
| `interruption_location_a` | `interruption_location_a` | 直接映射 |
| `interruption_location` | `interruption_location` | M2M 复制 |
| `cutover_longitude` | `interruption_longitude` | 直接映射 |
| `cutover_latitude` | `interruption_latitude` | 直接映射 |
| `cutover_reason` | `fault_details` | 拼接来源说明 |
| `resource_type` | `resource_type` | 直接映射 |
| `resource_owner` | `resource_owner` | 直接映射 |
| `cable_route` | `cable_route` | 直接映射 |
| `maintenance_mode` | `maintenance_mode` | 直接映射 |
| `handling_unit` | `handling_unit` | 直接映射 |
| `contract` | `contract` | 直接映射 |
| `line_supervisor` | `line_manager` | 直接映射 |
| `implementation_unit` / `cutover_contact` | `handler` | 拼接为处理人建议值 |
| `is_timeout` | `timeout` | `no -> True`，`yes -> False`，`pending -> True` |
| `timeout_reason` | `timeout_reason` | 直接映射 |
| `comments` | `comments` | 拼接保留 |

### 4.2 默认值规则

| `OtnFault` 字段 | 默认值 |
| --- | --- |
| `fault_category` | `FaultCategoryChoices.FIBER_BREAK` |
| `urgency` | `UrgencyChoices.LOW` |
| `interruption_reason` | `cable_rectification` |
| `interruption_reason_detail` | 根据报备状态，默认 `planned_reporting` |
| `fault_status` | 有 `completed_at` 则 `closed`，否则 `processing` |
| `first_report_source` | 留空 |
| `dispatch_time` | 留空 |
| `departure_time` | 留空 |
| `arrival_time` | 留空 |
| `repair_time` | 留空 |
| `manager_reviewed` / `noc_reviewed` | 默认 `False` |

### 4.3 不映射字段

以下字段不从割接映射到故障：

| `CutoverTask` 字段 | `OtnFault` 字段 |
| --- | --- |
| `rectification_status` | `rectification_status` |
| `rectification_measures` | `rectification_measures` |
| `rectification_description` | `rectification_description` |
| `rectification_subject` | `rectification_subject` |
| `rectification_progress` | `rectification_progress` |
| `planned_completion_time` | `planned_completion_date` |
| `actual_completion_time` | `actual_completion_date` |
| `rectification_completion_description` | `rectification_completion_description` |

原因：割接整改闭环和故障整改闭环上下文不同，自动映射会污染故障复盘与统计。

### 4.4 影响业务映射

| `CutoverImpact` | `OtnFaultImpact` | 规则 |
| --- | --- | --- |
| `cutover_task` | `otn_fault` | 指向新生成的故障 |
| `service_type` | `service_type` | 直接映射 |
| `bare_fiber_service` | `bare_fiber_service` | 直接映射 |
| `circuit_service` | `circuit_service` | 直接映射 |
| `service_site_a` | `service_site_a` | 直接映射 |
| `service_site_z` | `service_site_z` | M2M 复制 |
| `business_impact` | `business_impact` | 直接映射 |
| `service_interruption_time` | `service_interruption_time` | 优先使用；为空则用故障起始时间 |
| `service_recovery_time` | `service_recovery_time` | 优先使用；为空则用故障恢复时间 |
| `comments` | `comments` | 拼接来源割接影响业务说明 |
| 无 | `secondary_faults` | 留空 |

## 5. 后端实现设计

### 5.1 服务层

新增文件：

```text
netbox_otnfaults/services/cutover_fault_generation.py
```

职责：
- 汇总割接生成故障的预填数据。
- 提供字段映射函数。
- 在事务中创建故障和影响业务。
- 保持视图薄逻辑，便于源码级测试。

建议函数职责：

| 函数 | 职责 |
| --- | --- |
| `build_fault_initial_data(cutover, user)` | 根据割接主记录和当前用户生成故障确认表单初始值，不写入数据库 |
| `create_fault_from_cutover(cutover, fault_data, user)` | 在事务中创建 `OtnFault`、复制 Z 端站点、创建所有 `OtnFaultImpact` |
| `_timeout_value(cutover)` | 将割接的 `yes/no/pending` 超时状态转换为故障的 Boolean `timeout` |
| `_fault_occurrence_time(cutover)` | 按 `planned_cutover_time -> started_at -> timezone.now()` 顺序选取故障起始时间 |

### 5.2 表单

新增：

```text
CutoverFaultGenerationForm
```

放在 `netbox_otnfaults/forms.py`。

表单基于 `forms.Form`，不直接继承 `OtnFaultForm`，避免把故障完整编辑页的复核、整改、供电字段带入确认页。

字段：
- `duty_officer`
- `fault_category`
- `urgency`
- `fault_status`
- `interruption_reason`
- `interruption_reason_detail`
- `cutover_report_status`
- `cutover_report_time`
- `fault_occurrence_time`
- `fault_recovery_time`
- `closure_time`
- `comments`

校验：
- `fault_recovery_time` 不得早于 `fault_occurrence_time`。
- `closure_time` 不得早于 `fault_occurrence_time`。
- 如果割接已经存在 `generated_faults`，表单提交应拒绝重复创建。

### 5.3 视图

新增视图：

```text
CutoverTaskGenerateFaultView
```

放在 `netbox_otnfaults/views.py`。

建议继承：

```python
class CutoverTaskGenerateFaultView(PermissionRequiredMixin, View):
    permission_required = 'netbox_otnfaults.add_otnfault'
```

行为：
- `GET`：构建预填表单和影响业务预览。
- `POST`：校验表单，创建故障与影响业务，跳转故障详情。

### 5.4 URL

## 6. 防重复与异常处理

默认策略：一个割接只能通过标准入口生成一条故障。

判断方式：

```python
existing_fault = cutover.generated_faults.order_by('-created').first()
```

如果存在：
- `GET` 页面展示已有故障链接和提示。
- `POST` 拒绝创建，提示用户已生成故障。

异常处理：
- 表单校验失败：留在确认页并显示字段错误。
- 影响业务创建失败：事务回滚，不留下半成品故障。
- 割接缺少 `interruption_location_a`：理论上模型必填；若历史脏数据为空，确认页阻止提交。

## 9. 验收标准

- 割接详情页能进入“生成故障”确认页。
- 确认页展示预填故障字段和将复制的影响业务。
- 用户确认后生成 1 条 `OtnFault`。
- 新故障 `source_cutover_task` 指向来源割接。
- 割接下每条 `CutoverImpact` 生成对应 `OtnFaultImpact`。
- 裸纤业务的 `service_site_z` 正确复制。
- 电路业务不会保留裸纤站点字段。
- `rectification_*` 字段不从割接映射到故障。
- 重复点击生成入口不会创建重复故障。
- 故障详情页能看到来源割接链接。
- 不修改 NetBox 核心目录。

## 10. 后续可选增强

以下内容不纳入第一阶段：
- 支持一个割接生成多条故障。
- 生成后回写割接状态。
- 自动根据 `cutover_report_status` 更精细地区分计划报备和非报备。
- 在 API 中提供专门的生成动作端点。
- 从故障详情页提供“回到来源割接并查看影响业务差异”的对照视图。
