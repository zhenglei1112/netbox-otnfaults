# 需求文档: 待协调割接小组件

## 1. 业务背景
在 OTN 故障与割接管理中，割接影响的业务需要由各业务主管进行协调审批。为了方便业务主管快速查看和处理需要自己协调的割接，我们需要在仪表盘增加一个“待协调割接”统计小组件。该小组件只显示当前登录用户负责的、仍处于待协调或未批准状态的割接影响业务数量，点击后能够一键跳转到割接影响业务列表中并只显示相关数据。

## 2. 功能设计

### 2.1 统计展示
* **组件类型**：数字信息卡片（2x2 大小），表现形式与“待复核故障”小组件完全一致。
* **图标**：
  * 当存在待协调的业务时，显示橘黄色（`#fd7e14`）的 `mdi-clock-alert-outline` 警告图标，并突出显示当前数量。
  * 当没有待协调的业务时，显示绿色（`#198754`）的 `mdi-clock-check-outline` 正常图标，数量显示为 `0`，副文本为“暂无待协调割接”。
* **标题**：
  * 有待协调时：“待协调割接”
  * 暂无待协调时：“暂无待协调割接”

### 2.2 数据筛选逻辑
统计的范围为“割接影响业务（`CutoverImpact`）”，过滤逻辑如下：
1. **协调状态（`coordination_status`）**：为 `pending`（待协调） 或 `unapproved`（未批准）。
2. **主管用户权限**：
   * 关联的裸纤业务（`bare_fiber_service`）的业务主管（`business_manager`）为当前登录用户；
   * 或关联的电路业务（`circuit_service`）的业务主管（`business_manager`）为当前登录用户。
3. 符合条件的总计数展示在卡片上。

### 2.3 跳转行为
* 点击小组件的卡片区域，跳转到割接影响业务的列表页面。
* 路由地址：`plugins:netbox_otnfaults:cutoverimpact_list`
* 请求参数：带有当前登录用户的筛选参数 `my_pending_coordination=<user_id>`。
* 列表展示：仅展示过滤后的割接影响业务。
