# 大屏地图视口与缩放优化（消除右侧留白与视口截断）需求

## 1. 需求背景与问题分析
- **当前表现**：在全国态势图大屏上，即使经度向西平移后，中国最西端（新疆）依然容易发生溢出被截断（看起来被左面板“挡住”），而右侧依然留有大面积黑影。
- **根本原因**：
  1. **物理视口挤压**：地图容器并不是全屏，在物理上被左右侧栏挤压在中间，导致其中间可用部分的**长宽比变得比较窄**。
  2. **扁平图形与容器宽高比冲突**：中国版图是一个极其扁平宽阔的形状。如果 `mapZoom` 缩放级别过大（之前为 `4.2` / `4.3`），在当前较窄的视口容器下，中国版图的横向物理宽度会**超出地图容器的物理宽度**。从而导致最左端的新疆和最右端的部分在超出容器边缘后被 `overflow: hidden` 自然裁剪（看起来“显示不去”）。
- **优化方案**：
  - 调整地图默认缩放级别 `mapZoom`：将兜底默认缩放级别由原来的 `4.3` 下调至 **`4.0`**。整体微调收拢，使整个中国版图完整收纳在较窄的中间窗口中，避免两侧边缘被容器剪切，从而把新疆完全“拉出”露出。
  - 调整地图默认中心点 `mapCenter`：将默认中心经度微调为 **`104.0`**（较原始的 `108.9` 往西平移 `4.9` 度）。由于整体缩放级调小后版图已经完整，因此不需要过度的西移，小幅西偏即可使视觉重心在除去左右边栏后完美居中，填补右侧海洋黑影。
  - 调整地图默认倾斜角 `mapPitch`：保持下调后的 **`32°`** 视角以极大减弱近大远小的透视膨胀畸变。
