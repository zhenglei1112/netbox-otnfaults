# PRD - 割接管理省份自动分配线路主管

## 1. 变更背景
为了提高割接任务录入的效率，且保证其使用体验与故障模型一致，当用户在割接编辑页面（`cutovertask_edit.html`）中选择或更改省份（`province`）字段时，系统需要自动根据省份匹配并设置对应的线路主管（`line_supervisor`）。

## 2. 业务映射规则
线路主管与省份关键字的映射规则（与故障模型保持一致，使用最新维护的映射）：
- **冯鑫源**：甘肃、宁夏、青海、新疆、山东、陕西、内蒙古、浙江、四川、江西
- **姜川**：广东、湖南、福建、重庆、贵州、云南、海南
- **李立彬**：北京、天津、河北、河南、山西、广西
- **孙振伟**：湖北、安徽、江苏、上海
- **李小涛**：黑龙江、吉林、辽宁

## 3. 实现方案
1.  **前端监听**：在割接编辑页面（`cutovertask_edit.html`）的内嵌脚本中增加 `initManagerLogic` 逻辑。
2.  **获取选中的省份名称**：通过 TomSelect 实例安全获取选中的省份文本。
3.  **匹配主管**：根据省份文本匹配到对应主管姓名。
4.  **接口查询**：通过 AJAX 接口请求 `/api/users/users/?q=[主管姓名]` 在系统中查询匹配的用户 ID。
5.  **自动预填**：动态向线路主管（`line_supervisor`）的 TomSelect 实例中添加选项并设置为选中状态。

## 4. 验证计划
1.  **手动测试**：打开割接新增/编辑表单，分别选择“浙江”、“广东”、“北京”、“湖北”、“黑龙江”等省份，确认“线路主管”字段会自动填充为对应的人员。
