# 需求文档：故障统计明细同组重复故障浅底色高亮展示

在故障统计模块的故障明细列表中，已支持将重复发生的故障打上“重复”紫色标签并支持“按重复排序”聚类排列。

**优化目标**：
为了在列表中进一步加强重复故障组别的视觉聚合，要求：
1. **同组重复故障使用相同的背景底色**展示；
2. **不同的重复故障组，使用不同的浅色底色**展示以示区隔；
3. **全部采用浅色体系**，不刺眼，且必须自适应系统的亮暗（Light/Dark）主题。
4. 非重复故障保持默认背景，不做底色突出。

---

## 1. 配色方案设计（浅色体系）

挑选 6 种高雅的莫兰迪浅色作为循环调色盘。由于需要支持系统的黑白主题（Dark/Light Mode）切换：
- **亮色主题**：底色需极浅（防止和前置历史退色重合或干扰文字阅读），背景透明度设定在 `0.08` 左右。
- **暗色主题**：底色在深色卡片背景上必须有足够的区分度，透明度设定在 `0.18` 左右。

### 调色盘参数表：

| 类名 | 对应颜色 | 亮色背景 (RGBA) | 暗色背景 (RGBA) |
| :--- | :--- | :--- | :--- |
| `repeat-group-color-0` | 柔粉/红 | `rgba(220, 53, 69, 0.08)` | `rgba(220, 53, 69, 0.18)` |
| `repeat-group-color-1` | 浅蓝 | `rgba(13, 110, 253, 0.08)` | `rgba(13, 110, 253, 0.18)` |
| `repeat-group-color-2` | 浅绿 | `rgba(40, 167, 69, 0.08)` | `rgba(40, 167, 69, 0.18)` |
| `repeat-group-color-3` | 浅紫 | `rgba(111, 66, 193, 0.08)` | `rgba(111, 66, 193, 0.18)` |
| `repeat-group-color-4` | 浅黄/橙 | `rgba(253, 126, 20, 0.08)` | `rgba(253, 126, 20, 0.18)` |
| `repeat-group-color-5` | 浅青 | `rgba(32, 201, 151, 0.08)` | `rgba(32, 201, 151, 0.18)` |

此外，前续回溯历史故障的淡化底色类 `.statistics-preceding-fault-row` 包含 `background-color` 属性。为了能透出重复组的底色，同时保留 `opacity: 0.55` 达成的文字退色效果，我们将 preceding 样式的默认灰色背景设定为“在不存在重复组底色类时方才生效”：
```css
.statistics-preceding-fault-row:not([class*="repeat-group-color-"]) {
    background-color: var(--statistics-surface-subtle, rgba(0, 0, 0, 0.02)) !important;
}
```

---

## 2. 交互与渲染逻辑

在前端故障明细数据渲染（`renderDetailsTable` 和 `renderBranchCompanyDetailsTable`）阶段：
1. **底色分配**：过滤完列表数据后，统一执行底色分配算法。对所有 `is_repeat` 为真的条目执行时间段合并判定（同 A 端、Z 端交集且 60 天内），若聚类后组员数量多于 1 个，则为该组分配专属的 `repeat-group-color-X` 类别。
2. **HTML 拼接**：如果在列表行对象 `item` 上被挂载了 `repeatGroupClass`，则拼接 HTML 时把该 CSS 类附加到 `<tr>` 的 `class` 中。

---

## 3. 验证计划

- **暗黑模式兼容性**：切换后台的黑夜模式，验证高亮底色和黑色背景能和谐相衬。
- **列表独立排序验证**：在时间排序和重复排序模式下，检查是否都能精确将同一组故障标记为同一种浅背景色。
- **单元测试回归**：运行单元测试及语法测试，确保新增的前端函数和样式规则无破坏性。
