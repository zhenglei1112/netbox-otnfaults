# 诊断未指定站点的路径脚本方案

描述：我们需要编写一个 Netbox Custom Script，自动检查 `OtnPath` 模型中，其A/Z端站点名叫"未指定"的路径。然后提取其空间几何数据 `geometry`，通过第一个和最后一个坐标点，与系统内其余真实站点进行距离对比计算，给出最近站点的名称。脚本还要列出路径的描述等用于对照输出报告。

## User Review Required
> [!IMPORTANT]
> 1. 判断“站点未指定”的方式：目前拟定的是通过判断关联的 `site_a.name` 或 `site_z.name` 包含字符串 `"未指定"`。请确认您的站点的确是这样命名的。
> 2. 获取端点坐标：目前是通过查询 `OtnPath.geometry`（GeoJSON LineString格式）中的首尾坐标 `coordinates[0]` 和 `coordinates[-1]` 并进行球面距离（Haversine）计算。

## Proposed Changes

### 自定义脚本
#### [NEW] [diagnose_route_service.py](file:///d:/Src/netbox-otnfaults/scripts/diagnose_route_service.py)
建立脚本逻辑为：
- 定义继承自 `Script` 的 `CheckUnknownSitesScript`。
- 获取包含“未指定”站点的路径列表（`path.site_a` 或 `path.site_z` 包含“未指定”）。
- 计算与所有带坐标物理站点的距离，输出符合要求的最短距离物理站点及其距离、当前路径名称和描述。
- 输出为脚本执行报告日志（`log_success`, `log_info`, `log_warning`）。

## Open Questions
