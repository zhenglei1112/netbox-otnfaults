# 故障统计物理故障页影响等级指标 PRD

## 1. 背景

故障统计模块的“物理故障”页需要增加“故障和异常事件（按影响程度划分等级）”指标。该指标用于按 I-V 类和割接实施口径汇总统计周期内的故障与异常事件，口径来自业务统计表格，并需要落到当前插件已有模型与统计页实现。

当前系统已经具备以下基础能力：

- `OtnFault.fault_category` 可区分光缆中断、空调故障、供电故障、设备故障、光缆劣化、光缆抖动。
- `OtnFault.source_cutover_task` 可识别由割接生成的计划性故障。
- `OtnFault.is_suspended` 与 `fault_status=挂起` 可识别挂起故障。
- `OtnFault.power_fault_impact` 可识别供电故障是否导致设备脱管。
- `OtnFaultImpact` 可记录裸纤业务、电路业务是否中断。
- `CircuitService` 当前缺少“重要业务”结构化字段，无法判断“重要电路业务中断”。
- 空调故障、设备故障当前缺少结构化字段，无法判断是否达到 I 类故障影响程度。

因此本需求不采用纯前端展示或硬编码分类，而是补齐必要字段后由后端统一计算统计等级。

## 2. 目标

### 2.1 功能目标

在“故障统计 > 物理故障”页新增一组指标，展示当前统计周期内：

- 总次数
- I 类和 II 类故障（阻断故障）
- I 类故障
- II 类故障
- III 类事件（重要隐患类事件）
- IV 类事件（一般隐患类事件）
- V 类（挂起的故障）
- 割接实施

同时在“指标说明”弹窗中补充完整统计口径，确保页面展示和业务定义一致。

### 2.2 数据目标

补齐三个结构化判定字段：

- `OtnFault.ac_fault_is_class_i`：空调故障是否 I 类故障。
- `OtnFault.device_fault_is_class_i`：设备故障是否 I 类故障。
- `CircuitService.is_important`：电路业务是否重要业务。

字段命名可在实现时根据项目现有命名风格微调，但语义必须保持一致。

### 2.3 非目标

本次不引入独立的“影响等级”人工覆写模型。

本次不改变现有故障分类字段含义，不拆分 `fault_category`。

本次不重构故障统计主接口的大结构，只在现有物理故障统计 payload 中新增等级指标。

本次不新增 Netbox 核心 API，也不修改 `netbox/` 核心目录。

## 3. 字段需求

### 3.1 空调故障 I 类判定字段

在 `OtnFault` 增加布尔字段：

- 字段语义：该空调故障是否引发网络设备不能正常工作。
- 默认值：`False`。
- 适用范围：仅当 `fault_category=空调故障` 时参与统计。
- 表单展示：故障编辑页在空调故障相关字段区域展示。
- 详情展示：故障详情页在空调故障信息中展示。
- 列表筛选：第一阶段可不加入列表筛选；如后续有排查需求再补。

### 3.2 设备故障 I 类判定字段

在 `OtnFault` 增加布尔字段：

- 字段语义：该设备故障是否产生实际影响，不属于“设备部件冗余配置，未产生任何影响”的情况。
- 默认值：`False`。
- 适用范围：仅当 `fault_category=设备故障` 时参与统计。
- 表单展示：故障编辑页在设备故障相关字段区域展示。
- 详情展示：故障详情页在设备故障信息中展示。
- 列表筛选：第一阶段可不加入列表筛选；如后续有排查需求再补。

### 3.3 电路业务重要字段

在 `CircuitService` 增加布尔字段：

- 字段语义：该电路业务是否属于重要电路业务。
- 默认值：`False`。
- 表单展示：电路业务编辑页展示。
- 详情展示：电路业务详情页展示。
- 列表展示：电路业务列表可展示为徽标或布尔图标。
- API 序列化：通过插件 REST API 输出该字段。
- 过滤器：支持按是否重要业务过滤，便于后续统计校验。

## 4. 统计口径

### 4.1 统计周期

所有指标默认跟随物理故障页顶部统计周期和省份过滤。

周期取值复用现有 `filter_type` 与日期选择逻辑，包括年、半年、季度、月、周。

省份过滤复用现有物理故障页省份多选控件。

### 4.2 计划性割接排除

“总次数”和 I-V 类指标均不计入计划性割接故障。

计划性割接故障识别方式：

- `OtnFault.source_cutover_task IS NOT NULL` 的故障视为计划性割接生成故障。

这些故障不进入总次数，也不进入 I-V 类等级统计。

割接次数在“割接实施”指标中单独统计。

### 4.3 挂起排除与 V 类口径

挂起故障识别方式沿用现有统计口径：

- `fault_status=挂起`，或
- `is_suspended=True`。

I 类、II 类、III 类、IV 类均不统计挂起故障。

V 类单独统计挂起故障数量。

V 类不排除故障分类，但排除计划性割接故障，避免割接生成故障同时进入 V 类。

### 4.4 总次数

总次数统计当前周期内非计划性割接故障总数。

总次数包含：

- 光缆中断
- 空调故障
- 供电故障
- 设备故障
- 光缆劣化
- 光缆抖动
- 挂起故障

总次数不包含：

- `source_cutover_task IS NOT NULL` 的计划性割接故障。

该口径与当前物理页“故障总数”已有口径不同。新增等级指标中的“总次数”必须按本 PRD 口径计算，不应复用当前排除光缆劣化、光缆抖动的 `overall_total_count`。

### 4.5 I 类故障

I 类故障统计当前周期内非计划性、非挂起，且满足以下任一条件的故障。

#### 4.5.1 光缆中断 I 类

`fault_category=光缆中断`，并且存在以下任一业务中断影响：

- 关联 `OtnFaultImpact` 中存在裸纤业务中断。
- 关联 `OtnFaultImpact` 中存在电路业务中断，且该 `CircuitService.is_important=True`。

裸纤业务中断判断沿用现有影响业务字段：

- `service_type=裸纤业务`
- `business_impact=业务中断`

重要电路业务中断判断：

- `service_type=电路业务`
- `business_impact=业务中断`
- `circuit_service.is_important=True`

补充规则：

如果因 3 路由保护，3 起物理线路同时中断并引发同一电路故障，则这 3 起光缆中断均计入 I 类故障。统计单位是物理故障记录，不是业务影响记录。

#### 4.5.2 供电故障 I 类

`fault_category=供电故障`，并且：

- `power_fault_impact=设备脱管`。

#### 4.5.3 空调故障 I 类

`fault_category=空调故障`，并且：

- `ac_fault_is_class_i=True`。

#### 4.5.4 设备故障 I 类

`fault_category=设备故障`，并且：

- `device_fault_is_class_i=True`。

### 4.6 II 类故障

II 类故障统计：

- `fault_category=光缆中断`
- 非计划性割接
- 非挂起
- 不满足 I 类光缆中断条件

即“不属于 I 类故障的光缆中断”。

### 4.7 I 类和 II 类故障（阻断故障）

I 类和 II 类故障统计为：

- I 类故障数量 + II 类故障数量

该指标不含挂起。

### 4.8 III 类事件（重要隐患类事件）

III 类事件统计非计划性、非挂起，且不属于 I 类故障的以下故障：

- 空调故障
- 供电故障
- 设备故障

具体口径：

- 空调故障：`fault_category=空调故障` 且 `ac_fault_is_class_i=False`。
- 供电故障：`fault_category=供电故障` 且未导致设备脱管。
- 设备故障：`fault_category=设备故障` 且 `device_fault_is_class_i=False`。

### 4.9 IV 类事件（一般隐患类事件）

IV 类事件统计非计划性、非挂起的：

- 光缆抖动
- 光缆劣化

### 4.10 V 类（挂起的故障）

V 类统计当前周期内非计划性割接，且满足挂起条件的故障数量。

挂起条件：

- `fault_status=挂起`，或
- `is_suspended=True`。

### 4.11 割接实施

割接实施统计本期间实施的割接次数。

统计对象：

- `CutoverTask`

统计时间字段：

- 使用 `started_at`。
- 若 `started_at` 为空，可不计入“已实施”口径。

统计条件：

- `started_at >= period_start`
- `started_at < period_end`

该指标不从 `OtnFault.source_cutover_task` 反推，以避免“一个割接生成多起故障”导致割接实施次数被放大。

## 5. 页面展示需求

### 5.1 展示位置

在“故障统计 > 物理故障”页新增“故障和异常事件（按影响程度划分等级）”区块。

展示位置：

- 放在“总体情况”之后，“裸纤业务中断情况”之前。

理由：

- 等级统计是总体情况的进一步拆解。
- 它比光缆中断、裸纤业务中断更偏管理口径，应靠近总体概览。

### 5.2 指标卡片

新增区块展示 8 个指标：

- 总次数
- I 类和 II 类故障（阻断故障）
- I 类故障
- II 类故障
- III 类事件（重要隐患类事件）
- IV 类事件（一般隐患类事件）
- V 类（挂起的故障）
- 割接实施

卡片风格复用现有统计页 `statistics-strip-card` 体系，保持 NetBox 原生 Bootstrap 5 风格。

每个指标展示：

- 数值
- 单位：次 / 起
- 与上周期趋势对比
- 可点击下钻标识

### 5.3 指标说明

在现有“指标说明”弹窗中新增“故障和异常事件等级”说明段。

说明内容必须覆盖：

- 总次数不含计划性割接，但包含光缆抖动、劣化。
- I/II 类故障均不含挂起。
- I 类故障四类判定规则。
- II 类故障为不属于 I 类的光缆中断。
- III 类事件为除 I 类外的空调、供电、设备故障。
- IV 类事件为光缆抖动、光缆劣化。
- V 类为挂起故障。
- 割接实施按 `CutoverTask.started_at` 统计。
- 多条物理线路同时中断引发同一重要业务中断时，按物理故障记录逐条计入 I 类。

## 6. 下钻需求

### 6.1 故障类指标下钻

点击以下指标时，下方物理故障明细表按对应等级过滤：

- 总次数
- I 类和 II 类故障
- I 类故障
- II 类故障
- III 类事件
- IV 类事件
- V 类

新增明细接口过滤参数：

- `impact_level_total=true`
- `impact_level=class_i_ii`
- `impact_level=class_i`
- `impact_level=class_ii`
- `impact_level=class_iii`
- `impact_level=class_iv`
- `impact_level=class_v`

实际参数名称可实现时调整，但前后端必须通过测试固定。

### 6.2 割接实施下钻

点击“割接实施”时跳转或筛选到割接任务列表。

第一阶段可使用链接跳转，不要求在物理故障明细表中混排割接任务。

跳转筛选应表达当前统计周期：

- `started_at >= period_start`
- `started_at < period_end`

如果当前 NetBox 列表过滤 URL 无法稳定表达该范围，则第一阶段只提供到割接任务列表的入口，并在 PRD 验收中标注为“可接受降级”。

## 7. 后端聚合设计

### 7.1 新增聚合 helper

在 `statistics_views.py` 中新增独立 helper，例如：

```python
def _compute_physical_impact_level_summary(
    faults: list[OtnFault],
    cutover_count: int,
) -> dict[str, int]:
    ...
```

该 helper 应返回：

```python
{
    "total": 0,
    "class_i_ii": 0,
    "class_i": 0,
    "class_ii": 0,
    "class_iii": 0,
    "class_iv": 0,
    "class_v": 0,
    "cutover_implemented": 0
}
```

同时返回上周期同结构数据，用于前端趋势展示。

### 7.2 查询优化

为了判定光缆中断 I 类，需要预取影响业务：

- `OtnFaultImpact`
- `circuit_service`
- `bare_fiber_service`

统计查询必须集中预取，避免逐条故障查询影响记录。

实现时需要注意：

- 统计单位是 `OtnFault`，不是 `OtnFaultImpact`。
- 一条故障存在多个影响业务时，只能计入一次 I 类故障。
- 同一故障同时影响裸纤和重要电路时，也只计入一次 I 类故障。

### 7.3 明细过滤复用

新增等级判定逻辑应抽成可复用函数，避免主统计和明细接口各自实现一套规则。

提供统一判定函数：

```python
def _impact_level_for_fault(fault: OtnFault) -> str | None:
    ...
```

返回值：

- `class_i`
- `class_ii`
- `class_iii`
- `class_iv`
- `class_v`
- `None`

`None` 表示该故障为计划性割接故障或不进入等级统计。

## 10. 验收标准

- 物理故障页展示新增等级指标区块。
- 所有等级指标跟随顶部统计周期和物理页省份过滤。
- 总次数不含计划性割接，但包含光缆抖动、光缆劣化。
- I/II/III/IV 类均不含挂起故障。
- V 类只统计挂起故障。
- 光缆中断 I 类可由裸纤业务中断或重要电路业务中断触发。
- 空调、设备故障可通过新增字段进入 I 类。
- 供电故障可通过设备脱管进入 I 类。
- 割接实施按割接任务实际开始时间统计，不因一个割接生成多起故障而重复计数。
- 指标说明与后端统计口径一致。
- 新增字段完成模型、表单、详情、API、FilterSet 和迁移接入。
- 所有新增或调整测试通过。

## 11. 风险与后续扩展

### 11.1 风险

- 空调和设备故障的 I 类字段依赖人工录入，历史数据默认 `False`，上线后历史统计可能低估 I 类数量。
- 重要电路业务字段上线前，历史电路业务默认非重要，历史周期的 I 类光缆中断可能低估。
- 如果 `OtnFaultImpact` 记录不完整，光缆中断 I 类判定会受到影响。

### 11.2 后续扩展

- 增加“影响等级人工覆写”字段，用于特殊场景强制指定 I-V 类。
