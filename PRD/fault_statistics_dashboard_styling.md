# 光缆网故障统计看板UI配色与样式重构需求 (PRD)

## 1. 背景与目标
在目前的 Netbox OTN 光通信可视化插件中，“故障统计”模块页面虽然满足了数据汇聚需求，但在 UI 设计上存在以下不足：
- **色彩凌乱**：包含大量沉闷的黑色加粗大字体数字，且下挂的各维度子指标使用了杂多的颜色（紫、绿、青、黄），缺乏视觉聚焦点与系统感。
- **与主站割裂**：未能充分吸收 Netbox 系统的原生主色系（靛青色 / Indigo / `#206bc4`），影响专业办公的高级感。

**本次需求目标**：使用全局性的思维对整个看板（包括内容字号、核心指标图元、Tab切换页、ECharts 图表）进行色彩和风格重塑，达成“美化、统一、全面覆盖、清爽的工业化办公风格”的美学目标。

## 2. 需求细则

### 2.1 统计卡片与核心数字 (数字看板)
- **主调统一**：整个页面的主体大数值（总体起数、中断总起数、时长、历时等）必须摒弃 `text-dark` (纯黑色)，统一启用全局主色系 `text-primary` (靛青色)。
- **维度对比数据颜色收敛**：
    - 针对平均历时下钻卡片（有效、日夜间、施工非施工等），摒弃以往强烈的颜色差异。
    - 客观数值部分统一采取 `text-primary` 或同色系克制变式表示，通过排版和标签区分维度，而不是依赖不同颜色。
- **趋势升降符语义规范**：
    - 任何导致可用性负向指标（如：起数增加 ⬆，总历时变长 ⬆）严格使用 `text-danger` (红)。
    - 任何导致可用性正向指标（如：故障减少 ⬇，历时变短 ⬇）严格使用 `text-success` (绿)。

### 2.2 图表配色规范 (ECharts)
- **直方图**：柱体颜色直接从紫红色切换为品牌靛蓝色（`#206bc4`）。
- **各类占比环/饼状图**：使用顺向同色系阶数（例如：深蓝 -> 浅蓝 -> 灰蓝组合），表示客观的从高到低数据占比，同时针对明确含有特殊属性（如：严重、未知）的内容可做适当灰色/中性色穿插。禁止使用红通通绿油油拼接而成的传统色环。

### 2.3 导航栏与操作控件
- **Tab 栏呈现**：强化非激活状态 `.nav-link` 的工业灰底色和字体，保证已激活标签采用清爽的 `#206bc4` （靛蓝）底边线和高亮，取消多余的内边距导致的臃肿感。
- **表单选型与按钮**：日期选择框与统计周期切换组，继承标准 Netbox / Bootstrap 5 的 Form control 样式，不过度自绘描边。

## 3. 技术落地方案概要 (对应 Implementation Plan)
本次落脚点在：
- `statistics_dashboard.html`: 清理模板中的 `text-dark`, `text-info`, `text-purple` 等非标准类。
- `statistics_dashboard.js`: 更新 JS 视图渲染逻辑里的 `buildFlexGroup` 与 echarts 的 `itemStyle.color` 调色盘选项，规范红绿箭头的计算逻辑。
- `statistics_dashboard.css`: 若需覆盖卡片边框阴影，注入少量精致化（`.shadow-sm` 级别的扁平阴影）的 UI 微调。
