# 部署验证工作流

每次完成代码修改后，必须执行以下验证步骤：

// turbo-all

## 1. 执行部署脚本

运行批处理文件以部署更改：

```powershell
d:\src\177.bat
```

## 2. 打开浏览器验证

使用浏览器打开系统网址，检查是否有错误：

- 访问地址: `http://192.168.30.177`
- 检查页面是否正常加载
- 检查控制台是否有 JavaScript 错误
- 检查网络请求是否有失败的请求

## 3. 报告结果

向用户报告验证结果：

- 部署脚本执行结果
- 页面加载状态
- 发现的任何错误
