# 故障月历紧凑网格样式设计

## 背景

故障月历小组件已经取消“每天最多 5 个故障点”的后端截断，但当前模板仍使用松散的 `flex-wrap` 圆点布局。同一天故障较多时，圆点会在较小的单元格里显得拥挤、对齐不稳，影响可读性。

## 目标

- 保留“一个圆点代表一个故障”的现有表达方式
- 在不恢复截断的前提下，让多故障日期的点位排列更整齐
- 保持当前 NetBox 风格和现有颜色语义不变
- 仅做月历模板样式优化，不引入新交互

## 方案

### 推荐方案：紧凑网格

将每天的故障点容器从自由换行的 `flex` 布局改为固定列数的小网格布局：

- 点位容器使用紧凑 grid 排列，保证多行时仍然整齐对齐
- 单个圆点尺寸略微缩小，间距同步缩小，提升单位面积承载能力
- 日期区域与点阵区域视觉分层，给点阵预留稳定高度，减少不同日期之间的跳动感

## 范围

本次仅修改：

- `netbox_otnfaults/templates/netbox_otnfaults/inc/dashboard_calendar_widget.html`

本次不修改：

- 后端月历数据结构
- 故障分类颜色映射
- tooltip、弹窗、展开、悬停明细等交互

## 验证

- 增加源码级测试，约束月历模板改为紧凑网格布局
- 运行定向测试，确认样式结构已落入模板

## 风险与控制

- 若列数过多，少量故障的日期可能显得过于稀疏
- 若圆点过小，颜色辨识度会下降

因此实现时采用“轻量压缩”而不是极限压缩，优先保证整齐和可辨识，而不是追求单元格内塞入最多点数。
