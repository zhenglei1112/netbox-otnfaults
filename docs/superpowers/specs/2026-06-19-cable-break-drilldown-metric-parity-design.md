# 光缆中断指标与下钻明细口径统一设计

## 目标

确保“光缆中断情况”区域内所有可点击指标与故障明细使用相同的数据范围和筛选条件，并按指标类型清晰解释指标值与明细起数、历时汇总之间的关系。

## 统一基础范围

光缆中断卡片、主要原因、光缆属性、历时频数和各省份故障统计统一限定为当前统计周期、当前页面选择的省份范围、故障类型为光缆中断、`is_suspended=False` 且故障状态不是挂起。省份图不得绕过当前页面的省份筛选使用全局数据。

## 指标分类与下钻解释

- 起数指标：明细当期行数必须等于指标起数。
- 历时指标：明细行数表示涉及故障起数，服务端汇总的原始历时之和必须等于指标历时。
- 平均指标：展示明细起数、累计历时及“累计历时 / 起数”的结果。
- 超时率：列出历时不少于 4 小时的故障，并展示“超时故障起数 / 光缆中断总起数”。
- P50/P90：指标值是历时阈值，不是明细数量；说明中明确阈值和下钻比较方向。

## 重复中断

重复中断卡片使用 KPI 口径：当前故障存在 60 天内更早发生、A 端相同且至少一个 Z 端相同的光缆故障。明细下钻只返回满足该 KPI 条件的当前周期故障，不用 UI 双向关联口径扩大当前故障数量。

## 数据流与验证

前端点击时携带统一 `detail_scope=cable_break`，同时保存指标类型和值。明细接口在应用筛选后，基于未舍入的故障历时返回 `summary`；前端使用服务端汇总生成说明，表格继续使用 `results`。项目无 NetBox 运行环境，因此使用现有统计测试、Python 编译检查和 JavaScript 语法检查验证。
